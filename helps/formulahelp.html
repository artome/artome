dsv2db can deal with building some variables obtained from numerical expressions from others variables at the same line regardeless of them being stored in database or not.


The mathematical parsing is made by The C++ Mathematical Expression Toolkit Library (ExprTk) https://www.partow.net/programming/exprtk/index.html.


The variables always, named equal to variables of the header (but time), are enclosed by @headervariavle@


example:


signal=@volt@*@int@/5.


will evaluate the signal value by multiplying the volt value by the value of int and divides by 5.


volt and int must be in the header field but not the signal.

But the signal must be one of the columns


Parsing mathematical expressions can be a very time expensive process and there are sometimes simples operations that are required just imagine one wants to average the value of 50 columns eg. columns 3 till 52

one has a tailor mode that works directly with columns values.


vmean=SF3-52;0.02 will produce the average value of the 53 coulmn values.


variable=SF#i-#f;fact where #i is the first column and #f the last column will add all the values of the columns #i to #f and multiply the result by fact.


If the columns to add are not continuous one can use commas

variable=SF5,9,10,50;1. will sum the columns 5,9,10 and 50.
